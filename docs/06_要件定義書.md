# 要件定義書

## 1. システム概要

飲み会の幹事（ホスト）がAIエージェントと対話して飲み会を企画し、
参加者への連絡・やりとりをAIに委任するWebアプリケーション。
データはSupabase (PostgreSQL)に保存し、別端末からもアクセス可能。

## 2. アクター定義

| アクター | 説明 |
|----------|------|
| ホスト | 飲み会を企画・管理する人。AIと対話しPJを作成する |
| 参加者 | URLを受け取り、AIとやりとりして参加可否を回答する人 |
| AIエージェント | ホストの代理として参加者とやりとりするAI (gpt-4o-mini) |

## 3. 画面一覧

| # | 画面名 | アクター | 概要 |
|---|--------|----------|------|
| S1 | トップ画面 | ホスト | APIキー入力、新規PJ作成、飲み会一覧への導線 |
| S2 | ホストチャット画面 | ホスト | AIと対話（左サイドバーで参加者スレッドも閲覧） |
| S3a | 飲み会一覧画面 | ホスト | PJカード一覧（これから/過去に分類、削除可能） |
| S3b | ダッシュボード画面 | ホスト | 参加者管理・通知・精算・一斉通知送信 |
| S4 | 参加者チャット画面 | 参加者 | AIと対話して参加可否回答・グループ通知受信 |
| S5 | 期限切れ画面 | 参加者 | 招待リンク期限切れ時の案内 |

## 4. 画面遷移図

```
[S1: トップ]
  |-- 「新しい飲み会を作る」→ [S2: ホストチャット]
  |-- 「飲み会一覧を見る」→ [S3a: 飲み会一覧]

[S2: ホストチャット]
  |-- ←ボタン → [S3a: 飲み会一覧]
  |-- 「管理」ボタン → [S3b: ダッシュボード]
  |-- サイドバー: AIチャット / 参加者スレッド切替

[S3a: 飲み会一覧]
  |-- PJカードタップ → [S3b: ダッシュボード]
  |-- ×ボタン → PJ削除

[S3b: ダッシュボード]
  |-- ←ボタン → [S3a: 飲み会一覧]
  |-- 「チャット」ボタン → [S2: ホストチャット]（履歴復元済み）

[招待URL] → 有効期限チェック
  |-- OK → [S4: 参加者チャット]
  |-- NG → [S5: 期限切れ画面]
```

## 5. 機能要件

### F1: 飲み会PJ作成

| 項目 | 内容 |
|------|------|
| トリガー | トップ画面で「新しい飲み会を作る」ボタン押下 |
| 処理 | PJをDBに新規作成 → ホストチャット画面に遷移 |
| 備考 | スロットはヒアリング後に人数分作成（PJ作成時には作らない） |

### F2: ホスト-AIチャット（フォーマット入力方式）

**ヒアリングフロー:**
1. AIが空のフォーマットテンプレートを提示:
```
━━━━━━━━━━━━━━━━
🍻 飲み会名:
📅 日時:
📍 場所/店名:
💰 予算（1人あたり）:
👥 人数:
📝 備考:
━━━━━━━━━━━━━━━━
```
2. ホストがフォーマットを埋めて送信（複数行対応: Shift+Enterで改行）
3. AIが確認 →「✅ 要件確定」→ 確定フォーマットを出力（コピーボタン付き）
4. **人数分のスロットをDBに自動作成**
5. 参加者名をヒアリング →「✅ 参加者登録: 名前1, 名前2, ...」

**運用中の対応:**
- 変更通知:「✅ 変更通知: {内容}」→ 全参加者にブロードキャスト
- 中止:「✅ 中止」→ 全参加者に中止通知
- 参加/キャンセル等の📢通知がホストチャットに自動投稿

### F3: 左サイドバー（会話スレッド一覧）

| 項目 | 内容 |
|------|------|
| PC | 280px幅の固定サイドバー |
| モバイル | ☰ハンバーガーメニューでオーバーレイ表示 |

**サイドバー内容:**
- 🤖 AIチャット（ホスト-AI対話）: 入力可能
- 👤 各参加者名: 閲覧のみ（やりとり履歴の確認）
- 各アイテムに最新メッセージのプレビュー（30文字）
- 選択中のスレッドはハイライト表示

### F4: 参加者登録・スロット管理

**スロット作成:**
- ヒアリングで「4人」→4枠、「10人」→10枠を自動作成
- デフォルト4枠（人数未指定時）

**ダッシュボード操作:**
- 「名前を入力」: 空きスロットに参加者名を登録（仮名可）
- 名前の編集: 入力欄で直接変更
- 「＋ 枠を追加」: 空きスロットを1つ追加
- 「－ 最後の枠を削除」: 最後のスロットを削除（確認付き）
- 「URLコピー」: 参加者の招待リンクをクリップボードにコピー
- 「催促」: 未回答者にリマインドメッセージ投稿
- 「再発行」: 期限切れ招待リンクを再発行

**招待リンク:**
- 形式: `{アプリURL}#join/{PJ_ID}/{参加者ID}`
- 有効期限: 発行から24時間
- 期限切れ時: S5（期限切れ画面）を表示

### F5: ダッシュボード

**タブ構成:**
1. **参加者タブ**: スロット一覧（ステータス・電話番号・招待期限・キャンセル料）
2. **通知タブ**: 通知ログ時系列表示（未読バッジ付き）
3. **精算タブ**: 精算サマリ（見込み合計・実費入力・割り勘計算・キャンセル料一覧）

**一斉通知:**
- テキスト入力+送信ボタン
- `member_id='group'`でDBに保存
- ホストチャットにも📢メッセージ追加
- 参加者チャットにグループ通知として表示

**通知対象イベント:**
参加表明 / 不参加表明 / キャンセル / アレルギー申告 / 遅刻連絡 / 追加参加申請 / 電話番号登録 / 招待期限切れ / 催促送信 / 招待再発行 / 一斉通知

### F6: 参加者-AIチャット

**初回フロー:**
1. AIが飲み会詳細をフォーマットで表示
2. 参加可否を確認
3. 参加表明後、電話番号をヒアリング

**参加者ができること:**
- 参加/不参加の回答 → `✅ 参加OK` / `✅ 不参加`
- キャンセル → `✅ キャンセル` + 送金先案内（三菱UFJ銀行 渋谷支店 普通1234567 ノミカイタロウ）
- アレルギー・食事制限の申告 → `✅ アレルギー: {内容}`
- 遅刻連絡 → `✅ 遅刻: {内容}`
- 追加参加者の申請 → `✅ 追加参加: {内容}`
- 電話番号の回答 → `✅ 電話番号: {番号}`

**グループ通知:**
- 幹事からの一斉通知を「📢 幹事から: {内容}」として紫系スタイルで表示
- 個別メッセージとグループ通知を時系列でマージ表示

**ホストへの通知連動:**
- 参加/不参加/キャンセル等のイベント発生時:
  - notifications テーブルに記録
  - ホストチャット(member_id=null)に📢メッセージを投稿
  - ダッシュボードの通知バッジを更新

### F7: 飲み会一覧

**分類:**
- 🔜 これからの飲み会: 日付が未来 or パース不能
- 📁 過去の飲み会: 日付が過去

**操作:**
- PJカードタップ → ダッシュボードに遷移
- ×ボタン → PJ削除（confirm後、projects/members/chat_messages/notifications全削除）

### F8: チャット履歴の永続化

- 全チャット履歴はSupabase (chat_messages) に保存
- ページリロード・再訪問時にDBから復元
- ホストチャット: `resumeHostChat()` でDB復元→画面描画
- 参加者チャット: `startMemberChat()` でDB復元→画面描画

## 6. データ設計

### テーブル構造

**projects**
| カラム | 型 | 説明 |
|--------|-----|------|
| id | text PK | PJ ID (pj-xxxxx) |
| created_at | timestamptz | 作成日時 |
| status | text | ヒアリング中/参加者登録中/運用中/中止 |
| details | jsonb | {name, date, place, budget, headcount, note} |

**members**
| カラム | 型 | 説明 |
|--------|-----|------|
| id | text PK | 参加者ID (m-xxxxx) |
| project_id | text FK | PJ ID |
| slot | int | スロット番号 |
| name | text | 参加者名（空=空席） |
| status | text | 空席/未回答/参加/不参加/キャンセル/期限切れ |
| invited_at | timestamptz | 招待日時 |
| expires_at | timestamptz | 招待期限 |
| cancel_fee | int | キャンセル料 |
| phone | text | 電話番号 |
| last_updated | timestamptz | 最終更新日時 |

**chat_messages**
| カラム | 型 | 説明 |
|--------|-----|------|
| id | bigint PK | 自動採番 |
| project_id | text FK | PJ ID |
| member_id | text | null=ホストチャット, m-xxx=参加者個別, 'group'=一斉通知 |
| role | text | user / assistant |
| content | text | メッセージ内容 |
| created_at | timestamptz | 送信日時 |

**notifications**
| カラム | 型 | 説明 |
|--------|-----|------|
| id | bigint PK | 自動採番 |
| project_id | text FK | PJ ID |
| type | text | join/decline/cancel/allergy/late/plus_one/phone/expired/nudge/reissue/broadcast |
| message | text | 通知メッセージ |
| read | boolean | 既読フラグ |
| created_at | timestamptz | 通知日時 |

### データフロー

```
[projects] 1 ←→ N [members]
[projects] 1 ←→ N [chat_messages]
  └ member_id=null  → ホスト-AI対話
  └ member_id=m-xxx → 参加者個別チャット
  └ member_id=group → 一斉通知
[projects] 1 ←→ N [notifications]
```

## 7. 非機能要件

| 項目 | 内容 |
|------|------|
| レスポンシブ | モバイルファースト。サイドバーはモバイルでハンバーガーメニュー化 |
| データ保存 | Supabase (PostgreSQL)。APIキーのみlocalStorage |
| AI | OpenAI API (gpt-4o-mini) |
| ホスティング | GitHub Pages（静的HTML配信） |
| URL共有 | ホストがURLをコピーし、既存SNS/メッセンジャーで手動送付 |
| RLS | ハッカソンデモのため全テーブルallow all |

## 8. 技術上の制約

1. **OpenAI APIキー**: フロントエンドに露出（デモ用割り切り）。端末ごとにlocalStorage保存
2. **RLS無効**: 全テーブルに対してallow allポリシー。本番ではユーザー認証+RLS必須
3. **リアルタイム性**: Supabaseのリアルタイム機能は未使用。手動リロードで最新状態を取得
4. **リマインド**: サーバーレスのため自動リマインドは不可。ホストが手動で催促ボタンを押す
