# CLAUDE.md 解説

## CLAUDE.md とは？
プロジェクトルートに置く設定ファイル。Claude Codeは会話開始時にこのファイルを
自動的に読み込み、記載されたルールに従って開発を行う。

## 役割
1. **プロジェクト概要の共有** - Claudeにアプリの目的と背景を伝える
2. **技術ルールの定義** - 使用する技術スタック、コーディング規約を統一
3. **フォルダ構成の明示** - ファイルの配置ルールを定義
4. **コマンドの記録** - ビルドやプレビューの起動方法を記載

## 本プロジェクトでの設定内容

| 項目 | 設定値 | 理由 |
|------|--------|------|
| 構成 | 単一HTMLファイル | ハッカソン向けにシンプルに保つ |
| 外部ライブラリ | CDN経由のみ | ファイル数を最小限に |
| データ保存 | localStorage | サーバー不要で動作 |
| UI言語 | 日本語 | 国内ハッカソン向け |
| localStorage接頭辞 | `nomikai-app-` | キー名の衝突を防止 |

## 編集のタイミング
- アプリの方向性が決まったとき → 概要を更新
- 新しい技術ルールを追加したいとき → 開発ルールに追記
- フォルダ構成を変更したとき → 構成図を更新

## 親ディレクトリの CLAUDE.md との関係
Claude Codeは**親ディレクトリの CLAUDE.md も読み込む**。
現在、親ディレクトリ（ハッカソン/）には CLAUDE.md は無いが、
もし配置すれば全サブプロジェクト共通のルールを定義できる。
