## 2. Supabaseプロジェクト作成

### 2.1 プロジェクト新規作成

1. https://supabase.com にログインする
2. 「New Project」をクリック
3. 以下を入力して「Create new project」をクリック:
   - **Name**: 任意（例: `nomikai-agent`）
   - **Database Password**: 任意のパスワード（控えておく）
   - **Region**: `Northeast Asia (Tokyo)` を推奨
4. プロジェクト作成完了まで数分待つ

### 2.2 接続情報の取得

1. Supabase Dashboard左メニュー「Settings」>「API」を開く
2. 以下の2つの値を控える:
   - **Project URL**: `https://xxxxxxxx.supabase.co` 形式の値
   - **anon (public) key**: `eyJ...` で始まる長い文字列

### 2.3 テーブル作成（SQL実行）

1. Supabase Dashboard左メニュー「SQL Editor」を開く
2. 「New query」をクリック
3. `docs/supabase-setup.sql` の内容を全てコピーして貼り付ける
4. 「Run」をクリック
5. エラーが表示されないことを確認する

作成されるテーブルは以下の4つ:

| テーブル名 | 用途 |
|------------|------|
| `projects` | 飲み会PJ情報 |
| `members` | 参加者（スロット）情報 |
| `chat_messages` | チャット履歴 |
| `notifications` | 通知ログ |

### 2.4 membersテーブルへの phone カラム確認

`members` テーブルに `phone` カラムが存在することを確認する。SQLに含まれていない場合は以下を実行:

```sql
ALTER TABLE members ADD COLUMN IF NOT EXISTS phone text;
```

## 3. GitHubリポジトリ作成

### 3.1 リポジトリ新規作成

1. https://github.com にログイン
2. 「New repository」をクリック
3. 以下を設定:
   - **Repository name**: `nomikai-app`（任意）
   - **Public** を選択（GitHub Pages無料利用のため）
   - 「Add a README file」にチェック
4. 「Create repository」をクリック

### 3.2 ローカルにクローン

```bash
git clone https://github.com/<ユーザー名>/nomikai-app.git
cd nomikai-app
```

### 3.3 フォルダ構成の作成

```bash
mkdir -p app docs .github/workflows .claude
```

以下の構成でファイルを配置する:

```
nomikai-app/
├── CLAUDE.md
├── index.html               # app/index.html のコピー（GitHub Pages用）
├── .claude/
│   └── launch.json
├── .github/workflows/
│   └── pages.yml
├── app/
│   └── index.html           # アプリ本体
├── docs/
│   └── supabase-setup.sql
└── .gitignore
```

### 3.4 GitHub Pages の有効化

1. GitHubリポジトリページ「Settings」タブを開く
2. 左メニュー「Pages」をクリック
3. 以下を設定:
   - **Source**: 「GitHub Actions」を選択
4. 「Save」をクリック

### 3.5 GitHub Actions ワークフローの配置

`.github/workflows/pages.yml` に以下の内容を配置する:

```yaml
name: Deploy to GitHub Pages
on:
  push:
    branches: [main]
permissions:
  contents: read
  pages: write
  id-token: write
jobs:
  deploy:
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/configure-pages@v5
      - uses: actions/upload-pages-artifact@v3
        with:
          path: '.'
      - id: deployment
        uses: actions/deploy-pages@v4
```

## 4. 環境変数・接続情報の設定

### 4.1 Supabase接続情報の設定箇所

`app/index.html` の JavaScript 部分（268行目付近）に以下の定数がある:

```javascript
const SUPA_URL = 'https://xxxxxxxx.supabase.co';
const SUPA_KEY = 'sb_publishable_xxxxxxxxxx';
const sb = supabase.createClient(SUPA_URL, SUPA_KEY);
```

手順2.2で取得した値に書き換える:

- `SUPA_URL`: Project URL の値
- `SUPA_KEY`: anon (public) key の値

### 4.2 ルートの index.html 同期

`app/index.html` を編集したら、ルートの `index.html` にも同じ内容をコピーする:

```bash
cp app/index.html index.html
```

### 4.3 OpenAI APIキーについて

OpenAI APIキーはソースコードに埋め込まない。アプリのトップ画面（S1）でユーザーが入力し、`localStorage` に端末ごとに保存される仕組みとなっている。

## 5. ローカル開発環境のセットアップ

### 5.1 Python http.server による起動

プロジェクトルートで以下を実行:

```bash
python -m http.server 3001 -d app
```

ブラウザで http://localhost:3001 にアクセスして動作確認する。

### 5.2 Claude Code のプレビュー機能

`.claude/launch.json` が配置されている場合、Claude Code のプレビュー機能から `nomikai-app` を選択して起動できる。ポートは `3001` で設定済み。

```json
{
  "version": "0.0.1",
  "configurations": [
    {
      "name": "nomikai-app",
      "runtimeExecutable": "python",
      "runtimeArgs": ["-m", "http.server", "3001", "-d", "app"],
      "port": 3001
    }
  ]
}
```

> **注意**: `runtimeExecutable` のパスは環境に合わせて変更すること。Pythonにパスが通っていれば `"python"` で動作する。

## 6. デプロイフロー

### 6.1 通常のデプロイ手順

1. `app/index.html` を編集する
2. ルートの `index.html` に同期する:
   ```bash
   cp app/index.html index.html
   ```
3. 変更をコミットしてプッシュする:
   ```bash
   git add .
   git commit -m "変更内容の説明"
   git push origin main
   ```
4. GitHub Actions が自動で実行される
5. リポジトリの「Actions」タブでデプロイ成功を確認する
6. 公開URL（`https://<ユーザー名>.github.io/nomikai-app/`）にアクセスして確認する

### 6.2 デプロイの仕組み

```
app/index.html を編集
        ↓
cp app/index.html index.html（手動同期）
        ↓
git push origin main
        ↓
GitHub Actions (.github/workflows/pages.yml) が起動
        ↓
リポジトリ全体を GitHub Pages にアップロード
        ↓
https://<ユーザー名>.github.io/<リポジトリ名>/ で公開
```

## 7. トラブルシューティング

### GitHub Pages にデプロイされない

| 原因 | 対処法 |
|------|--------|
| Pages の Source が「GitHub Actions」になっていない | Settings > Pages で「GitHub Actions」を選択する |
| ワークフローファイルが正しい場所にない | `.github/workflows/pages.yml` に配置されているか確認する |
| Actions が無効になっている | Settings > Actions > General で「Allow all actions」を選択する |
| ルートに `index.html` がない | `cp app/index.html index.html` を実行し忘れていないか確認する |

### Supabase に接続できない

| 原因 | 対処法 |
|------|--------|
| `SUPA_URL` / `SUPA_KEY` が間違っている | Supabase Dashboard の Settings > API から正しい値をコピーし直す |
| テーブルが作成されていない | SQL Editor で `supabase-setup.sql` を再実行する |
| RLSポリシーが未設定 | SQLの `create policy` 文が実行されているか確認する。`allow all` ポリシーが4テーブル全てに必要 |

### AIチャットが動作しない

| 原因 | 対処法 |
|------|--------|
| OpenAI APIキーが未入力 | トップ画面でAPIキーを入力する |
| APIキーが無効 | OpenAI Dashboardで有効なキーか確認する。残高があるか確認する |
| ブラウザの開発者ツールでエラー確認 | F12 > Console タブでエラーメッセージを確認する |

### ローカル開発サーバーが起動しない

| 原因 | 対処法 |
|------|--------|
| Pythonがインストールされていない | https://www.python.org からインストールする |
| ポート3001が使用中 | 別のポート番号に変更する（例: `python -m http.server 8080 -d app`） |
| `-d app` オプションでエラー | Python 3.7以上が必要。`python --version` で確認する |

### 招待リンクが機能しない

| 原因 | 対処法 |
|------|--------|
| URLのハッシュ形式が正しくない | `{公開URL}#join/{PJ_ID}/{参加者ID}` の形式になっているか確認する |
| 招待期限が切れている | ダッシュボードの「再発行」ボタンで期限を更新する（発行から24時間有効） |
| 参加者のレコードがDBにない | ダッシュボードで参加者が正しく登録されているか確認する |

### データが消えた / 表示されない

| 原因 | 対処法 |
|------|--------|
| Supabaseプロジェクトが一時停止している | Supabase Dashboard でプロジェクトを再開する（無料プランは1週間非アクティブで停止） |
| 別のSupabaseプロジェクトに接続している | `SUPA_URL` と `SUPA_KEY` が正しいプロジェクトのものか確認する |
| ブラウザキャッシュの問題 | ハードリロード（Ctrl+Shift+R）を試す |
