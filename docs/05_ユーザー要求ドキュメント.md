# ユーザー要求ドキュメント

## 1. プロダクト概要

| 項目 | 内容 |
|------|------|
| プロダクト名 | 飲み会エージェント |
| コンセプト | 飲み会やご飯会の連絡・管理を代行するAIエージェント |
| ターゲットユーザー | 飲み会・食事会の幹事（ホスト）および参加者 |
| 公開URL | https://nekokawaiiperopero.github.io/nomikai-app/ |

## 2. 背景課題

### ホスト（幹事）が抱える課題

1. **やりとりの分散**
   - 飲み会の連絡がInstagram DM、LINE、口頭など複数チャネルに分散する
   - 誰とどのやりとりをしていたか忘れる

2. **個別対応の煩雑さ**
   - 参加者ごとに異なるやりとりが発生する（アレルギー確認、遅刻連絡、場所の質問など）
   - 全員に同じ情報を伝えるのにも1人ずつ連絡が必要

3. **情報の一元管理ができない**
   - 誰が参加で誰が不参加か、全体像を把握しづらい
   - キャンセル料の管理が手動

## 3. ユーザーストーリー

### ホスト（幹事）

> AIエージェントにフォーマットを埋めて送るだけで飲み会を作成し、
> 参加者への連絡・出欠確認・キャンセル料通知をAIに任せたい。
> ダッシュボードで全員の状況を一目で把握し、
> 左サイドバーで各参加者とのやりとりも確認したい。
> 全員への一斉通知もワンクリックで送りたい。

### 参加者

> 送られてきたURLを開くだけで、飲み会の詳細を確認し、
> AIに参加可否を伝え、電話番号を教え、質問があればAIに聞きたい。
> 幹事からのグループ通知もチャット内で受け取りたい。

## 4. 利用フロー

```
[ホスト]                      [AIエージェント]              [参加者]
   |                                |                         |
   |-- (1) PJ作成 ----------------->|                         |
   |-- (2) フォーマットで要件入力 ->|                         |
   |       (人数分の枠を自動作成)   |                         |
   |-- (3) 参加者名を登録 -------->|                         |
   |                                |                         |
   |   ダッシュボードでURLコピー → LINE/Insta等で参加者に送付 |
   |                                |                         |
   |                                |<-- (4) URLアクセス -----|
   |                                |--- 飲み会詳細を表示 --->|
   |                                |<-- 参加可否を回答 ------|
   |                                |<-- 電話番号を回答 ------|
   |                                |<-- 質問・やりとり ----->|
   |                                |                         |
   |<-- (5) ダッシュボードで確認    |                         |
   |<-- (6) サイドバーでやりとり確認|                         |
   |<-- (7) 📢全員に通知を送信      |                         |
   |                                |--- グループ通知 ------->|
   |                                |                         |
   |   参加者がキャンセル ←---------|<-- キャンセル申告 ------|
   |                                |--- 送金先を案内 ------->|
   |<-- ホストチャットに📢通知      |                         |
```

### (1) プロジェクト作成
- トップ画面で「新しい飲み会を作る」→ AIチャット画面に遷移

### (2) フォーマットで要件入力
- AIが空のフォーマットテンプレートを提示
- ホストがフォーマットを埋めて送信（複数行入力対応）
- AIが内容を確認し、要件確定
- **入力された人数分のスロット**をダッシュボードに自動作成

### (3) 参加者名の登録
- AIチャットで名前を伝える、またはダッシュボードから直接入力
- 仮名（「友人A」等）でも登録可能、後から変更可能
- 枠の追加・削除もダッシュボードから可能

### (4) 参加者のURLアクセス
- 招待リンクの有効期限は24時間（期限切れ時は再発行可能）
- AIが飲み会詳細をフォーマットで表示し、参加可否を確認
- 参加表明後、電話番号をヒアリング
- 幹事からのグループ通知もチャット内に表示

### (5) ダッシュボード管理
- 参加者一覧（ステータス・電話番号・招待期限・キャンセル料）
- 通知ログ（参加/不参加/キャンセル/アレルギー/遅刻等）
- 精算サマリ（実費入力→割り勘計算）
- 全員への一斉通知送信

### (6) サイドバーでやりとり確認
- 左サイドバーにAIチャット＋各参加者スレッド一覧
- 各スレッドの最新メッセージをプレビュー表示
- 参加者スレッドをクリックしてやりとりを閲覧

### (7) キャンセル対応
- 参加者がキャンセル → AIが送金先（口座情報）を案内
- キャンセル料は予算の100%
- ホストチャットに📢通知、ダッシュボードに反映

## 5. 飲み会一覧管理

- 「これからの飲み会」「過去の飲み会」に自動分類
- 飲み会の削除（関連データ全削除）
- 一覧からPJ選択→ダッシュボード詳細に遷移

## 6. 技術構成

| 項目 | 内容 |
|------|------|
| フロントエンド | HTML / CSS / JavaScript（単一HTMLファイル） |
| AI | OpenAI API (gpt-4o-mini) |
| データベース | Supabase (PostgreSQL) |
| ホスティング | GitHub Pages |
| 外部ライブラリ | Supabase JS SDK (CDN) |

## 7. やらないこと（スコープ外）

- LINE / Instagram DM 連携
- ユーザーアカウント登録・ログイン
- 店舗検索・予約連携
- プッシュ通知・自動リマインド
- ユーザー認証・マルチテナント
