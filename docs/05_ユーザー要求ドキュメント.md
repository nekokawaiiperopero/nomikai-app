# ユーザー要求ドキュメント

## 1. プロダクト概要

| 項目 | 内容 |
|------|------|
| プロダクト名 | 飲み会エージェント（仮） |
| コンセプト | 飲み会やご飯会の連絡を代行するAIエージェント |
| ターゲットユーザー | 飲み会・食事会の幹事（ホスト）および参加者 |

## 2. 背景課題

### ホスト（幹事）が抱える課題

1. **やりとりの分散**
   - 飲み会の連絡がInstagram DM、LINE、口頭など複数チャネルに分散する
   - 誰とどのやりとりをしていたか忘れる
   - 後から情報を探すのに時間がかかる

2. **個別対応の煩雑さ**
   - 参加者ごとに異なるやりとりが発生する（アレルギー確認、遅刻連絡、場所の質問など）
   - 全員に同じ情報を伝えるのにも1人ずつ連絡が必要
   - リマインドも手動で行う必要がある

3. **情報の一元管理ができない**
   - 誰が参加で誰が不参加か、全体像を把握しづらい
   - 参加者の回答状況がバラバラに散らばっている

## 3. ユーザーストーリー

### ホスト（幹事）のストーリー

> 私は飲み会の幹事として、AIエージェントと会話するだけで、
> 飲み会の詳細を決め、参加者全員への連絡・やりとりをAIに任せたい。
> そして全参加者の状況をダッシュボードで一目で確認したい。

### 参加者のストーリー

> 私は飲み会の参加者として、送られてきたURLを開くだけで、
> AIエージェントから飲み会の詳細を聞き、参加可否を伝え、
> 質問があればAIに聞きたい。

## 4. 利用フロー

### フロー全体像

```
[ホスト]                    [AIエージェント]                [参加者]
   |                              |                          |
   |-- (1) PJ作成 --------------->|                          |
   |-- (2) 要件を対話で確定 ----->|                          |
   |-- (3) 参加者情報を登録 ----->|                          |
   |                              |                          |
   |   ホストが参加者にURLを手動で送付（LINE/Insta/口頭等）  |
   |                              |                          |
   |                              |<-- (4) URLアクセス ------|
   |                              |--- 飲み会詳細を表示 ---->|
   |                              |<-- 参加可否を回答 -------|
   |                              |<-- 質問・やりとり ------>|
   |                              |                          |
   |<-- (5) ダッシュボードで状況確認                         |
```

### (1) プロジェクト作成
- ホストがアプリを開き、新しい飲み会PJを作成する

### (2) AIとの対話で要件確定
- AIがホストにヒアリングし、以下の情報を確定する：
  - 飲み会の名前（例：「金曜のサシ飲み」「チーム忘年会」）
  - 日時
  - 場所 / 店名
  - 予算感
  - 予定人数
  - その他メモ（ドレスコード、アレルギー対応等）

### (3) 参加者情報の登録
- ホストがAIに参加者の情報を伝える：
  - 名前
  - （連絡先はアプリ内チャットのためURL共有で代替）

### (4) 参加者のURL経由アクセス
- ホストが参加者にURLを手動で送付（既存のLINE/Instaなどで）
- 参加者がURLを開くと、Webチャット画面が表示される
- AIが飲み会の詳細を伝え、参加可否を確認する
- 参加者はAIに質問・変更依頼ができる

### (5) ダッシュボードでの状況確認
- ホストは全参加者の状況を一覧で確認できる：
  - 参加 / 不参加 / 未回答
  - やりとりの要約・ステータス

## 5. MVP方針（ハッカソンスコープ）

### やること
- アプリ内Webチャットで完結する（LINE/Insta連携なし）
- ホスト-AI対話による飲み会PJ作成
- 参加者向けURL生成とWebチャット
- ホスト向けダッシュボード

### やらないこと
- LINE Messaging API / Instagram API 連携
- ユーザーアカウント登録・ログイン機能
- 店舗検索・予約連携（ホットペッパー等）
- プッシュ通知・リマインド自動送信
- 複数飲み会の同時管理（将来機能）
