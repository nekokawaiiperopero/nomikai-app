# Claude Code 機能活用ガイド

本プロジェクトで活用できるClaude Codeの主要機能をまとめる。

---

## 1. CLAUDE.md（プロジェクト設定）
- **場所**: プロジェクトルート直下
- **用途**: 開発ルール、技術スタック、コーディング規約の定義
- **効果**: Claudeが毎回の会話でルールを自動参照し、一貫した開発を実現

## 2. Skills（スキル）
- **場所**: `.claude/skills/` ディレクトリ
- **用途**: 特定の作業パターンを再利用可能なテンプレートとして定義
- **本プロジェクトで利用中のスキル**:
  - `dev-standard` - 開発スタンダード（HTML/CSS/JS単一ファイル構成ルール）
  - `idea-framework` - アイデア出し・企画整理
- **呼び出し方**: Claudeが文脈に応じて自動で適用、または `/skill名` で明示的に呼び出し

## 3. Preview（プレビュー）
- **設定**: `.claude/launch.json`
- **用途**: ローカルサーバーを起動し、ブラウザでアプリを自動プレビュー
- **活用法**: Claudeがコード変更後に自動でスクリーンショットを撮り、表示を確認しながら開発
- **起動**: `preview_start` → ポート3001でapp/ディレクトリを配信

## 4. Memory（メモリ）
- **場所**: `~/.claude/projects/*/memory/`
- **用途**: セッションを跨いで情報を記憶
- **活用法**: プロジェクトの重要な決定事項や頻出パターンをClaudeが自動記録
- **注意**: CLAUDE.mdとは異なり、Claude自身が書き込む学習メモ

## 5. Agent（サブエージェント）
- **用途**: 複雑なタスクを並列で処理
- **活用例**:
  - コードベースの探索（Exploreエージェント）
  - 実装計画の設計（Planエージェント）
  - テスト実行

## 6. TodoWrite（タスク管理）
- **用途**: 開発タスクの進捗管理
- **活用法**: 複数ステップの作業をリスト化し、完了状況を可視化

## 7. Git連携
- **用途**: バージョン管理、コミット、ブランチ管理
- **活用法**: 作業の区切りでClaudeにコミットを依頼

---

## 推奨ワークフロー

```
1. アイデア出し    → idea-framework スキルを活用
2. 設計・計画      → EnterPlanMode で実装計画を策定
3. 実装            → dev-standard スキルに従いコーディング
4. プレビュー確認  → preview_start でブラウザ表示確認
5. 修正・改善      → スクリーンショットを見ながら調整
6. バージョン管理  → git commit で保存
```
