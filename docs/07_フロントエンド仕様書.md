## 7. AI プロンプト設計

### ホスト用システムプロンプト (`hostPrompt()`)

**動的パラメータ:**
- `pj.status`: 現在のプロジェクトステータス
- `pj.details`: 飲み会詳細（JSON）
- `members`: 登録済み参加者一覧（名前+ステータス）
- `notifs`: 直近5件の通知

**フェーズ構成:**

| フェーズ | トリガー | AIの動作 |
|----------|----------|----------|
| 1. ヒアリング | 初回会話 | フォーマットテンプレートを提示、埋めたら確認 |
| 2. 参加者登録 | 要件確定後 | 参加者名を受け取り登録 |
| 3. 運用中 | 登録完了後 | 変更通知・中止対応 |

### 参加者用システムプロンプト (`memberPrompt(member)`)

**動的パラメータ:**
- `member.name`: 参加者名
- `member.status`: 現在のステータス
- `pj.details`: 飲み会詳細全項目

**対応パターン:**
- 参加OK → `✅ 参加OK` → 電話番号ヒアリング
- 不参加 → `✅ 不参加`
- キャンセル → `✅ キャンセル（キャンセル料: ¥{予算}）` + 振込先案内
- アレルギー → `✅ アレルギー: {内容}`
- 遅刻 → `✅ 遅刻: {内容}`
- 追加参加 → `✅ 追加参加: {内容}`
- 電話番号 → `✅ 電話番号: {番号}`

**振込先情報（ハードコード）:**
- 三菱UFJ銀行 渋谷支店
- 口座番号: 普通 1234567
- 口座名義: ノミカイ タロウ

## 8. イベント検知ロジック（✅マーカー方式）

AI返答内の `✅` 文字列をパターンマッチングで検知し、対応するDB操作を実行する。

### ホスト側 (`processHostReply`)

| 検知パターン | 処理 |
|-------------|------|
| `✅` + `要件確定` | details抽出（正規表現で絵文字行をパース）→ DB更新、スロット作成 |
| `✅` + `参加者登録:` | ステータスを「運用中」に変更、名前をスロットに割当 |
| `✅` + `変更通知:` | 全参加者に変更内容をブロードキャスト |
| `✅` + `中止` | ステータスを「中止」に変更、全参加者に中止通知 |

**要件確定時の詳細抽出パターン:**

```
🍻\s*(.+)         → name
📅\s*日時[:：]\s*(.+)  → date
📍\s*場所[:：]\s*(.+)  → place
💰\s*予算[:：]\s*(.+)  → budget
👥\s*人数[:：]\s*(.+)  → headcount
📝\s*備考[:：]\s*(.+)  → note
```

**スロット数決定:** `headcount` から数値を抽出（`parseInt` + 非数値除去）。パース不能時は `DEFAULT_SLOTS` (4)。

**参加者名パース:** `参加者登録:` の後の文字列を `、` `,` 空白 全角空白で分割。敬称（さん/くん/ちゃん/氏）は除去。

### 参加者側 (`processMemberReply`)

| 検知パターン | 処理 |
|-------------|------|
| `✅ 参加OK` or `✅ 参加` (不参加/キャンセル/追加参加を含まない) | status → 参加、ホストに通知 |
| `✅ 不参加` | status → 不参加、ホストに通知 |
| `✅ キャンセル` | status → キャンセル、cancel_fee を予算額に設定、ホストに通知 |
| `✅ アレルギー:` | ホストに通知（ステータス変更なし） |
| `✅ 遅刻:` | ホストに通知（ステータス変更なし） |
| `✅ 追加参加:` | ホストに通知（ステータス変更なし） |
| `✅ 電話番号:` | phone フィールドを更新、ホストに通知 |

**ホストチャットへの通知連動:** イベント検知時に `dbAddChat(pjId, null, 'assistant', '📢 ' + hostMsg)` でホストチャット（member_id=null）に📢メッセージを投稿。

## 9. ルーティング

URLハッシュベースのルーティング。

| パターン | 動作 |
|----------|------|
| `#join/{pjId}/{memberId}` | 参加者チャット開始（`startMemberChat`） |
| その他 / なし | トップ画面表示 |

- `hashchange` と `load` の両イベントで `handleRoute()` を呼び出し
- 参加者URLの形式: `{origin}{pathname}#join/{pj-xxxxx}/{m-xxxxx}`
