# 開発ワークフロー

## 基本的な進め方

### Step 1: アイデア出し・企画整理
Claudeに「アイデアを考えて」と伝えると、`idea-framework` スキルが自動適用される。
- テーマに沿ったアイデアのブレスト
- 機能の優先度整理
- MVP（最小限の製品）の定義

### Step 2: 実装の指示
具体的な機能を伝えて実装を依頼する。

**指示の例:**
- 「割り勘計算機能を作って」
- 「投票機能を追加して」
- 「タイマー画面を作って」

`dev-standard` スキルが自動で適用され、以下のルールで生成される:
- 単一HTMLファイル（HTML + CSS + JS）
- 日本語UI
- レスポンシブ対応
- localStorage でデータ保存

### Step 3: プレビュー確認
Claudeが自動で `preview_start` を実行し、ブラウザ上で表示を確認する。
問題があれば「ボタンが小さい」「色を変えて」など自然言語で修正指示を出せる。

### Step 4: バージョン管理
区切りの良いところで「コミットして」と伝える。
Claudeが変更内容を確認し、適切なコミットメッセージを生成する。

---

## よく使う指示パターン

| やりたいこと | 指示例 |
|-------------|--------|
| 新機能の追加 | 「〇〇機能を作って」 |
| UI修正 | 「ボタンをもっと大きくして」 |
| バグ修正 | 「〇〇が動かないので直して」 |
| プレビュー | 「画面を見せて」 |
| コミット | 「コミットして」 |
| 計画を立てる | 「〇〇の実装計画を立てて」 |
| アイデア出し | 「飲み会で使える機能のアイデアを出して」 |

## ファイル命名規則

| ファイル種類 | 命名例 | 説明 |
|-------------|--------|------|
| メイン画面 | `index.html` | アプリのトップページ |
| 個別機能 | `vote.html` | 投票機能 |
| 個別機能 | `timer.html` | タイマー機能 |
| 個別機能 | `split-bill.html` | 割り勘機能 |
