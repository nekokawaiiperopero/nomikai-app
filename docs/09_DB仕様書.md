## 3. テーブル詳細定義

### projects（飲み会PJテーブル）

| カラム | 型 | NULL | デフォルト | 制約 | 説明 |
|--------|-----|------|-----------|------|------|
| id | text | NO | - | PRIMARY KEY | PJ ID（形式: `pj-{timestamp_base36}`） |
| created_at | timestamptz | YES | `now()` | - | 作成日時 |
| status | text | YES | `'ヒアリング中'` | - | PJステータス |
| details | jsonb | YES | `'{}'::jsonb` | - | 飲み会詳細情報 |

**details の JSON 構造:**

```json
{
  "name": "飲み会名",
  "date": "日時テキスト",
  "place": "場所/店名",
  "budget": "予算テキスト（例: 5000円）",
  "headcount": "人数テキスト（例: 4人）",
  "note": "備考"
}
```

- ヒアリング中は `{}` (空オブジェクト)
- 要件確定時にAI返答からパースして格納
- 全フィールドはテキスト型（数値のパースはフロントエンド側で行う）

**ID 生成ロジック:** `'pj-' + Date.now().toString(36)`

### members（参加者テーブル）

| カラム | 型 | NULL | デフォルト | 制約 | 説明 |
|--------|-----|------|-----------|------|------|
| id | text | NO | - | PRIMARY KEY | 参加者ID（形式: `m-{random_base36_6chars}`） |
| project_id | text | YES | - | FOREIGN KEY → projects(id) ON DELETE CASCADE | 所属PJ |
| slot | int | YES | - | - | スロット番号（1始まり） |
| name | text | YES | `''` | - | 参加者名（空文字列 = 空席） |
| status | text | YES | `'空席'` | - | 参加者ステータス |
| invited_at | timestamptz | YES | - | - | 招待日時 |
| expires_at | timestamptz | YES | - | - | 招待期限（invited_at + 24時間） |
| cancel_fee | int | YES | `0` | - | キャンセル料（円） |
| phone | text | YES | - | - | 電話番号 |
| last_updated | timestamptz | YES | - | - | 最終更新日時 |

**ID 生成ロジック:** `'m-' + Math.random().toString(36).slice(2, 8)`

**注意:** SQL定義には `phone` カラムが存在しないが、フロントエンドの `dbSaveMember` で `phone` プロパティを条件付きで upsert している。実運用では ALTER TABLE で追加する必要がある。

### chat_messages（チャット履歴テーブル）

| カラム | 型 | NULL | デフォルト | 制約 | 説明 |
|--------|-----|------|-----------|------|------|
| id | bigint | NO | IDENTITY (自動採番) | PRIMARY KEY | メッセージID |
| project_id | text | YES | - | FOREIGN KEY → projects(id) ON DELETE CASCADE | 所属PJ |
| member_id | text | YES | - | - | メッセージの所属先（後述の特殊値あり） |
| role | text | NO | - | NOT NULL | メッセージの送信者種別 |
| content | text | NO | - | NOT NULL | メッセージ本文 |
| created_at | timestamptz | YES | `now()` | - | 送信日時 |

### notifications（通知テーブル）

| カラム | 型 | NULL | デフォルト | 制約 | 説明 |
|--------|-----|------|-----------|------|------|
| id | bigint | NO | IDENTITY (自動採番) | PRIMARY KEY | 通知ID |
| project_id | text | YES | - | FOREIGN KEY → projects(id) ON DELETE CASCADE | 所属PJ |
| type | text | YES | - | - | 通知種別 |
| message | text | YES | - | - | 通知メッセージ |
| read | boolean | YES | `false` | - | 既読フラグ |
| created_at | timestamptz | YES | `now()` | - | 通知日時 |

**通知種別 (type) の値:**

| type 値 | 発生タイミング |
|---------|---------------|
| `join` | 参加者が参加を表明 |
| `decline` | 参加者が不参加を表明 |
| `cancel` | 参加者がキャンセル |
| `allergy` | 参加者がアレルギーを申告 |
| `late` | 参加者が遅刻を連絡 |
| `plus_one` | 参加者が追加参加を申請 |
| `phone` | 参加者が電話番号を登録 |
| `expired` | 参加者の招待が期限切れ |
| `nudge` | ホストが催促を送信 |
| `reissue` | ホストが招待を再発行 |
| `broadcast` | ホストが一斉通知を送信 |
| `change` | 飲み会詳細が変更された |
| `cancel_event` | 飲み会が中止された |

## 4. member_id の特殊値

`chat_messages.member_id` はメッセージの所属先を示す。値によって意味が異なる。

| 値 | 意味 | 用途 |
|----|------|------|
| `null` | ホスト-AIチャット | ホストとAIの対話メッセージ。📢通知メッセージもここに投稿される |
| `m-xxxxxx` (参加者ID) | 参加者個別チャット | 参加者とAIの個別やりとり |
| `'group'` (文字列) | グループ通知 | ダッシュボードからの一斉通知。全参加者のチャットに表示される |

### member_id 別のクエリパターン

```
ホストチャット取得:    .is('member_id', null)
参加者個別取得:        .eq('member_id', 'm-xxx')
グループ通知取得:      .eq('member_id', 'group')
参加者マージ表示取得:  個別 + group の2回クエリ → JS上で created_at ソート
```

### role の値

| 値 | 意味 |
|----|------|
| `'user'` | ユーザー（ホストまたは参加者）の発言 |
| `'assistant'` | AIの返答、📢通知メッセージ、グループ通知 |

## 6. RLS ポリシー一覧

| テーブル | ポリシー名 | 対象操作 | USING | WITH CHECK | 説明 |
|---------|-----------|---------|-------|------------|------|
| projects | `allow all projects` | ALL | `true` | `true` | 全操作許可 |
| members | `allow all members` | ALL | `true` | `true` | 全操作許可 |
| chat_messages | `allow all chat` | ALL | `true` | `true` | 全操作許可 |
| notifications | `allow all notifs` | ALL | `true` | `true` | 全操作許可 |

- 全テーブルで RLS は `ENABLE` されているが、ポリシーは `USING (true) WITH CHECK (true)` のため実質無制限
- ハッカソンデモ用の暫定設定

## 7. ステータス遷移図

### projects.status

```
┌──────────────┐    要件確定          ┌──────────────────┐    参加者登録        ┌──────────┐
│ ヒアリング中  │ ──────────────────> │ 参加者登録中      │ ──────────────────> │ 運用中    │
└──────────────┘    (✅ 要件確定)     └──────────────────┘   (✅ 参加者登録:)   └──────────┘
                                                                                    │
                                                                                    │ 中止
                                                                                    │ (✅ 中止)
                                                                                    ▼
                                                                                ┌──────┐
                                                                                │ 中止  │
                                                                                └──────┘
```

| ステータス | 説明 | 遷移トリガー |
|-----------|------|-------------|
| ヒアリング中 | 初期状態。AIがフォーマットを提示中 | PJ作成時に設定 |
| 参加者登録中 | 要件確定後、参加者名の入力待ち | AI返答に「✅ 要件確定」を検知 |
| 運用中 | 参加者登録完了。招待URL配布・回答受付中 | AI返答に「✅ 参加者登録:」を検知 |
| 中止 | 飲み会中止 | AI返答に「✅ 中止」を検知 |

### members.status

```
┌──────┐    名前登録          ┌──────────┐
│ 空席  │ ──────────────────> │ 未回答    │
└──────┘   (assignSlot /     └──────────┘
            参加者登録)            │
                                  │
                    ┌─────────────┼─────────────┬──────────────┐
                    │             │             │              │
                    ▼             ▼             ▼              ▼
               ┌──────┐    ┌──────────┐  ┌──────────┐  ┌──────────┐
               │ 参加  │    │ 不参加    │  │ 期限切れ  │  │ (再発行) │
               └──────┘    └──────────┘  └──────────┘  └──────────┘
                    │                         │              │
                    │                         │ reissue      │
                    │                         └──────────────┘
                    │                               → 未回答に戻す
                    │
                    ▼
               ┌──────────┐
               │ キャンセル │
               └──────────┘
```

| ステータス | 説明 | 遷移元 | 遷移トリガー |
|-----------|------|--------|-------------|
| 空席 | スロット作成直後。名前未登録 | - | スロット作成時のデフォルト |
| 未回答 | 名前登録済みだが参加可否未回答 | 空席 | `assignSlot()` または `processHostReply(参加者登録)` |
| 参加 | 参加を表明済み | 未回答 | AI返答に `✅ 参加OK` / `✅ 参加` を検知 |
| 不参加 | 不参加を表明済み | 未回答 | AI返答に `✅ 不参加` を検知 |
| キャンセル | 参加表明後にキャンセル | 参加 | AI返答に `✅ キャンセル` を検知 |
| 期限切れ | 招待リンクの有効期限超過 | 未回答 | `checkExpired()` または `startMemberChat()` で期限チェック |

**期限切れからの復帰:** `reissue()` で `status` を `'未回答'` に戻し、`invited_at` / `expires_at` を再設定。

## 8. テーブル作成SQL（完全版）

```sql
-- 飲み会エージェント Supabase テーブル定義
-- Supabase Dashboard > SQL Editor に貼り付けて実行

-- 飲み会PJテーブル
CREATE TABLE projects (
  id TEXT PRIMARY KEY,
  created_at TIMESTAMPTZ DEFAULT now(),
  status TEXT DEFAULT 'ヒアリング中',
  details JSONB DEFAULT '{}'::jsonb
);

-- 参加者テーブル
CREATE TABLE members (
  id TEXT PRIMARY KEY,
  project_id TEXT REFERENCES projects(id) ON DELETE CASCADE,
  slot INT,
  name TEXT DEFAULT '',
  status TEXT DEFAULT '空席',
  invited_at TIMESTAMPTZ,
  expires_at TIMESTAMPTZ,
  cancel_fee INT DEFAULT 0,
  phone TEXT,
  last_updated TIMESTAMPTZ
);

-- チャット履歴テーブル
CREATE TABLE chat_messages (
  id BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  project_id TEXT REFERENCES projects(id) ON DELETE CASCADE,
  member_id TEXT,
  role TEXT NOT NULL,
  content TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- 通知テーブル
CREATE TABLE notifications (
  id BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  project_id TEXT REFERENCES projects(id) ON DELETE CASCADE,
  type TEXT,
  message TEXT,
  read BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- インデックス
CREATE INDEX idx_members_project ON members(project_id);
CREATE INDEX idx_chat_project_member ON chat_messages(project_id, member_id);
CREATE INDEX idx_notifs_project ON notifications(project_id);

-- RLS有効化
ALTER TABLE projects ENABLE ROW LEVEL SECURITY;
ALTER TABLE members ENABLE ROW LEVEL SECURITY;
ALTER TABLE chat_messages ENABLE ROW LEVEL SECURITY;
ALTER TABLE notifications ENABLE ROW LEVEL SECURITY;

-- RLSポリシー（ハッカソンデモ用: 全操作許可）
CREATE POLICY "allow all projects" ON projects FOR ALL USING (true) WITH CHECK (true);
CREATE POLICY "allow all members" ON members FOR ALL USING (true) WITH CHECK (true);
CREATE POLICY "allow all chat" ON chat_messages FOR ALL USING (true) WITH CHECK (true);
CREATE POLICY "allow all notifs" ON notifications FOR ALL USING (true) WITH CHECK (true);
```

**注意:** 上記SQLは `docs/supabase-setup.sql` の内容に `phone TEXT` カラムと `last_updated TIMESTAMPTZ` カラムを追加した完全版。元のSQLにはこれらのカラムが欠落しているため、既存テーブルには以下のALTER文が必要:

```sql
-- 既存テーブルへの追加カラム（supabase-setup.sql に未記載のもの）
ALTER TABLE members ADD COLUMN IF NOT EXISTS phone TEXT;
-- last_updated は supabase-setup.sql に記載なし。フロントエンドでは利用している。
ALTER TABLE members ADD COLUMN IF NOT EXISTS last_updated TIMESTAMPTZ;
```
